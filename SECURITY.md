# Security Policy

## Scope

This project is a standalone utility script intended for local system maintenance and configuration management.

It is not affiliated with, endorsed by, or maintained by AnyDesk Software GmbH.

This project does not provide or support any mechanism for unauthorized access, bypassing licensing, or abuse of remote access software.

---

## Supported Versions

Security updates are provided only for the latest released version.

Older versions may contain known or unknown vulnerabilities and are not actively maintained.

Users are strongly encouraged to upgrade to the latest release.

---

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

Do not open public issues for sensitive vulnerabilities.

Instead, use:

* GitHub Security Advisories (preferred)
* Or open a private report if available

If none of the above is possible, open an issue labeled "security" **only if the risk is low and does not expose users**

Please include:

* A clear description of the issue
* Steps to reproduce
* Potential impact
* Suggested mitigation (if available)

---

## What is considered a security issue

Examples include:

* Execution of unintended commands
* Arbitrary code execution risks
* Data loss beyond expected behavior
* Privilege escalation issues
* Unsafe handling of system paths or files

---

## What is NOT considered a security issue

* Expected behavior of the script
* Changes made to third-party software configuration
* Misuse of the script outside its intended purpose
* Issues caused by unsupported environments or modified systems

---

## Disclaimer

This tool is provided "as is", without any warranty.

Users are responsible for reviewing and understanding the script before running it with administrative privileges.

---

## Responsible Use

This tool is intended for personal and authorized environments only.

Do not use this script on systems or networks where you do not have explicit permission.
